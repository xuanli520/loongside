# References Index

This directory holds supporting repository references that do not all belong to
the same audience.

It is intentionally narrower than a general docs section. These files support
contributor routing and repository workflow without becoming part of the normal
public docs landing path.

## Read This Index When

- you need contributor-direction material that is still useful to public
  contributors
- you need repository-native support docs that automation or maintainer workflow
  still depends on
- you want to know whether a reference belongs in `site/`, `docs/`, or outside
  the public docs flow

## Route By Audience

| If you need... | Start here | Why |
| --- | --- | --- |
| a contributor-facing map of where design, docs, QA, systems, or community help is most useful | [`contribution-areas.md`](contribution-areas.md) | this is the public contributor-direction reference |
| the maintainer-owned GitHub branch model, labels, intake wiring, and automation baseline | [`github-collaboration.md`](github-collaboration.md) | this is repository support material used by automation and maintainer workflow |

## What Belongs Here

- `contribution-areas.md` is normal contributor-facing reference material.
- `github-collaboration.md` remains repository-native support material.
- maintainer workflow references may stay in the repository when automation
  depends on them, without becoming part of the primary public docs path.

## Do Not Put Here By Default

- backlog notes, internal comparison material, or temporary authoring scratchpads
- duplicated Mintlify navigation pages or reader-facing tutorials
- maintainer-only planning bundles that should live outside the OSS docs flow
